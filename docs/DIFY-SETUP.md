# 🚀 Dify 启动指南

## 什么是 Dify？

**Dify** 是一个生产级的 AI 应用开发平台，类似 Java 的 Ruoyi 框架。

**特点**：
- ✅ **商业级**：很多公司直接基于 Dify 二次开发
- ✅ **可视化**：拖拽式 AI 应用编排
- ✅ **企业级**：权限管理、多租户、API 网关
- ✅ **134,806 Stars**：GitHub 最受欢迎的 AI 平台之一

---

## 📋 前置要求

- ✅ Docker 已安装
- ✅ Docker Compose 已安装
- ✅ 至少 4GB 内存
- ✅ 至少 10GB 磁盘空间

---

## 🚀 启动步骤

### 1. 进入目录

```bash
cd ~/ai-agent-learning/dify/docker
```

### 2. 启动服务

```bash
docker-compose up -d
```

**首次启动需要**：
- 下载 Docker 镜像（约 5-10 分钟）
- 初始化数据库（约 1-2 分钟）

### 3. 检查状态

```bash
# 查看容器状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

---

## 🌐 访问地址

启动成功后，访问：

| 服务 | 地址 | 说明 |
|------|------|------|
| **Dify 主页** | http://localhost/ | 主应用 |
| **安装向导** | http://localhost/install | 首次安装 |

**默认端口**：
- 控制台: 80
- API: 5001

---

## 📝 首次安装

### 1. 打开安装向导

```bash
open http://localhost/install
```

### 2. 设置管理员账号

- 用户名：admin
- 密码：自定义（至少 8 位）

### 3. 配置设置

- **语言**：选择中文
- **向量数据库**：默认 Weaviate（也可选择 Milvus）
- **LLM 模型**：选择 OpenAI 或其他

### 4. 完成安装

点击"开始使用"进入 Dify

---

## 🎯 核心功能

### 1. 可视化编排

创建 AI 应用的拖拽式界面：
- 添加节点（LLM、知识库、工具等）
- 连接节点定义流程
- 配置参数和提示词

### 2. 知识库

上传文档并创建知识库：
- 支持 TXT、MD、PDF、Word 等
- 自动分片和向量化
- 支持多种向量数据库

### 3. Agent 开发

创建智能 Agent：
- 自定义工具
- 多轮对话
- 记忆管理

### 4. API 服务

所有应用自动生成 API：
- RESTful API
- SDK 支持
- Webhook 集成

## 🛠️ 常用命令

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f api
docker-compose logs -f worker

# 查看特定服务
docker-compose ps

# 更新 Dify
git pull
docker-compose pull
docker-compose up -d
```

## 🔧 配置文件

### .env 文件

```bash
cd ~/ai-agent-learning/dify/docker
vi .env
```

**重要配置**：
- `CONSOLE_WEB_URL`: 控制台地址
- `CONSOLE_API_URL`: API 地址
- `DATASET_VECTOR_DB_TYPE`: 向量数据库类型
- `LLM`: 大语言模型配置

## 🐛 常见问题

### 1. 端口冲突

**问题**: 80 端口被占用

**解决**:
```bash
# 查看占用
lsof -i :80

# 修改端口
vi docker-compose.yaml
# 找到 "80:80" 改为 "8080:80"
```

### 2. 内存不足

**问题**: 容器频繁重启

**解决**:
- 增加 Docker 内存限制
- 或关闭不必要的服务

### 3. 无法访问

**问题**: localhost 无法打开

**解决**:
```bash
# 检查容器状态
docker-compose ps

# 查看日志
docker-compose logs -f
```
